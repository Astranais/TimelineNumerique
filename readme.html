Aide et informations sur ce jeu numérique de style Timeline

Règles de ce jeu

Le but est de réussir à placer les événements tirés aléatoirement au bon endroit dans la frise en cours de construction. Pour cela il faudra estimer au plus près possible la date de l'événement. Les événements affichés dans la frise forment des intervalles de dates. Si la date donnée par le joueur est située dans le même intervalle que la date réelle de l'événement, alors le point est accordé. Un point bonus est accordé si la date exacte est trouvée.

Concept et utilisation

Le concept du jeu est bien évidemment inspiré (repris) du jeu de société Timeline, adapté ici pour un usage pédagogique et non commercial pour l'enseignement ou la découverte ludique d'événements historiques d'une thématique particulière.

Ce jeu se présente sous la forme d'une page Web en accès hors ligne : un fichier HTML principal (index.html) accompagné d'un dossier d'images et de fichiers de style (styles.css), de gestion du jeu (script.js, PapaParse.js) ou de données (readme.html, events.csv). Il est fourni avec une base de données sur les événements de l'histoire de l'informatique rassemblés par la conceptrice de ce jeu numérique, mais chacun est libre d'utiliser ce jeu avec sa propre base de données pourvu qu'elle respecte le format adapté. Consulter la section Adaptation du jeu à ses besoins.

Étapes à suivre pour pouvoir jouer :

  1. D'ouvrir le fichier index.html avec un navigateur Web ;
  2. De sélectionner le thème (si vous souhaitez ne jouer qu'avec une partie restreinte des données du fichier .csv) et le nombre de joueurs ;
  3. De sélectionner le fichier event.csv contenant les données des événements à utiliser (c'est ici que vous pouvez sélectionner le fichier de votre conception du moment qu'il respecte le même format) ;
  4. Le 1er événement est placé sur la frise. Vous pouvez consulter sa description en appuyant sur la touche Entrée de votre clavier. Un nouvel appui sur cette touche affiche le nouvel événement à placer pour le joueur dont les scores sont encadrés ;
  5. Le joueur en cours entre alors une date qu'il estime la plus proche de la date de l'événement affiché et appui sur Entrée. Les points sont automatiquement mis à jour et l'événement placé sur la frise tandis que la description est affichée. Le joueur suivant peut jouer après avoir appuyé sur la touche Entrée ou le bouton Nouvel Événement.
  6. Le jeu s'arrête à votre convenance ou à l'épuisement des événements de la base de données. Pour effectuer une nouvelle partie, actualisez la page.
Pour fermer cette fenêtre d'aide, appuyer sur la touche Échap ou faite sortir la souris de l'encadré.

Adaptation du jeu à ses besoins

Vous pouvez utiliser cette interface pour jouer à partir de votre propre base de données d'événements. Il vous suffit de fournir un fichier de type CSV contenant les 7 colonnes suivantes séparées par un ; (la 1ère ligne de votre fichier doit contenir le titre de la colonne) :

Date
Nombre entier positif ou négatif, ne pas mettre de lettres ou de symboles autres.
Titre
Très courte description de l'événement (recto de la carte)
Description
(optionnelle) Description plus longue de l'événement à titre informatif (recto de la carte), permettant de préciser l'approximation éventuelle de la date, le contexte ou l'origine de l'illustration. Peut inclure des balises html pour le formatage comme <i></i> (italique) <b></b> (gras) ou <br/> (nouvelle ligne).
Catégorie_principale
(optionnelle mais recommandée) Un seul mot. Permet de faire une sélection d'une partie seulement des événements (utile pour les grosses bases de données).
Catégorie_secondaire
Idem, permet à un événement de rentrer dans 2 catégories.
Sélection
(optionnel) Colonne non encore exploitée dans le fichier JavaScript mais utilisée dans le logiciel de tableur pour la création du csv avec une partie seulement de sa base de données. Consulter le fichier Timeline_numérique.xlsx pour exemple.
Fichier_img
(optionnel) Nom et extension du fichier image contenu dans le dossier img.

Auteur et informations sur la conception

La conception de la version numérique de ce jeu ainsi que de la base de données contenant les événements ont été réalisées par Anaïs Kobsch, professeure d'informatique au lycée puis en classe préparatoire. Certains extraits de description des événements ont été directement repris de Wikipédia.
La plus grande partie de l'implémentation en Javascript a été réalisée par GPT-3.5 via l'interface Chat-GPT : le modèle de language a été dirigé précisément pas à pas pour réaliser les fonctions ou écouteurs d'événements Javascript à partir de la spécification ou de l'algorithme écrit en pseudo-code ou en python. Consulter le fichier script.js ou la discussion avec Chat-GPT pour en savoir plus. Cette discussion montre en particulier l'impasse dans laquelle on peut se trouver si on utilise Chat-GPT de façon trop vague, rechercher "Let's start over." dans la conversation (seconde occurrence) pour identifier le changement de stratégie dans l'utilisation de l'IA.
